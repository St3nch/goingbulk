# GoingBulk Writing Style Guide

## Purpose

This document defines how GoingBulk should sound and how it should discuss health, fitness, experiments, products, studies, sponsors, and personal results.

The goal is to be clear, honest, entertaining, and limitation-aware without sounding like medical advice, hype, or drama content.

## Core Voice

GoingBulk should sound:

- curious;
- direct;
- skeptical but fair;
- data-backed;
- honest about uncertainty;
- human and occasionally funny;
- humble around health claims;
- transparent about limitations;
- allergic to hype.

## Core Writing Rule

```text
Say what happened. Do not overclaim what caused it.
```

GoingBulk can describe personal data, protocols, and observations.

GoingBulk must not present N=1 personal results as proof, treatment guidance, or universal advice.

## Positioning Language

Good:

```text
I tracked what happened to me during this protocol.
```

```text
My data moved in the expected direction, but this does not prove causation.
```

```text
This was useful for me, under these conditions, with these limitations.
```

Bad:

```text
This supplement works.
```

```text
This lowered my blood pressure.
```

```text
You should take this.
```

```text
This proves the study was right.
```

## Forbidden or High-Risk Phrases

Avoid:

```text
proven to
cures
fixes
guaranteed
clinically proven for everyone
doctors hate this
one weird trick
you need to
this will
this caused
this treats
this prevents disease
everyone should
miracle
biohack that changes everything
```

Use instead:

```text
associated with
occurred during
my data showed
in my N=1 experiment
for me
under these conditions
may be worth discussing with a professional
was consistent with
was not consistent with
```

## Claim Safety Pattern

When discussing a result, use this pattern:

```text
During [date range/protocol], [metric] changed from [baseline] to [result].
This happened while [other context/confounders].
Because this is an N=1 personal experiment, it does not prove [intervention] caused the change.
```

Example:

```text
During the creatine period, my average bodyweight increased from 184.2 lb to 187.8 lb. This happened while I was also training consistently and eating in a calorie surplus. Because this is an N=1 personal experiment, it does not prove creatine caused the weight gain.
```

## Required Language By Content Type

### Experiment Pages

Must include:

- N=1 disclaimer;
- protocol summary;
- data sources;
- source/confidence labels;
- confounders;
- limitations;
- personal verdict;
- not medical advice note where relevant.

### Supplement Pages

Must include:

- consult a qualified professional language;
- warning about medication interactions where relevant;
- dose is personal protocol, not recommendation;
- sponsor/affiliate disclosure if applicable.

### Bloodwork / Blood Pressure / Glucose Pages

Must include prominent medical disclaimer.

Example:

```text
This is personal tracking data, not medical advice. Do not use this page to diagnose, treat, or manage a medical condition. If you have abnormal blood pressure, glucose, or bloodwork, talk to a qualified healthcare professional.
```

### Product Review Pages

Must include:

- disclosure status;
- usage window;
- personal experience vs measured data distinction;
- would-use-if-not-paid verdict;
- limitations;
- affiliate/sponsor disclosure before affiliate links.

## Research Review Style

Research should be summarized carefully.

Required fields:

```text
study title
authors/year
study design
population
sample size
intervention
control/comparison
duration
primary outcome
main finding
limitations
how my protocol differs
```

Do not cherry-pick only favorable evidence.

If evidence is mixed, say it is mixed.

## Study vs Personal Protocol Rule

Always separate:

```text
what the study tested
what a public claim says
what I personally did
what happened in my data
what conclusion is reasonable
```

Never blur these into one universal statement.

## No Drama Policy

GoingBulk tests claims and ideas, not people.

Good:

```text
I saw this claim, reviewed the source, and tested a practical version on myself.
```

Bad:

```text
This creator lied.
```

```text
I exposed this influencer.
```

```text
This video proves they are wrong.
```

If baited into unrelated political, personal, or drama topics, do not engage.

## Sponsor and Affiliate Language

Disclosures must be plain and near the relevant link or content.

Good:

```text
This page contains affiliate links. If you buy through them, I may earn a commission at no extra cost to you. This does not change the data, protocol, or conclusion.
```

For sponsored content:

```text
This product test was sponsored by [Brand]. The sponsor did not control the data, protocol conclusion, or final verdict. Sponsored status is disclosed because it may affect perceived bias.
```

If sponsor influence exists, disclose it.

## Product Verdict Language

Use:

```text
Would I still use this if nobody paid me?
Yes / No / Maybe / Still testing / Only in specific cases
```

If the answer changes later, update the review and explain why.

## Humor Guidelines

Humor is allowed when it does not undermine health seriousness.

Good humor targets:

- the creator's own struggles;
- boring consistency;
- data weirdness;
- project complexity;
- fitness internet nonsense as a broad phenomenon.

Avoid humor that:

- mocks medical conditions;
- mocks body size or appearance;
- attacks individuals;
- trivializes serious lab results;
- minimizes supplement or health risks.

## Correction Policy

If a factual error is found:

```text
Correction: In the original version, I misstated [specific fact]. This has been corrected on [date]. The correction [does/does not] affect the main conclusion.
```

Do not silently edit important factual mistakes.

## Title Guidelines

Good titles:

```text
I Tested a Study-Inspired Creatine Protocol for 90 Days
My Baseline Fitness Data Before Starting GoingBulk
Hume Pod vs DEXA: My 12-Week Body Composition Comparison
What My Cronometer Data Showed During My First Bulk
```

Avoid:

```text
This Supplement Changed Everything
Doctors Hate This Fitness Hack
I Proved Creatine Works
This YouTuber Was Wrong
```

## Pre-Publish Claim Checklist

Before publishing health/fitness claims:

```text
[ ] N=1 limitation stated?
[ ] Confounders listed?
[ ] Medical disclaimer present where needed?
[ ] No causation claim without evidence?
[ ] Source/confidence labels included?
[ ] Sponsor/affiliate disclosure included?
[ ] Research summarized accurately?
[ ] Personal result separated from general evidence?
[ ] No universal recommendation language?
[ ] Correction/update path clear?
```

## Core Rule

```text
Be interesting without becoming irresponsible.
```
